HTML

  • HTML = HyperText Markup Language
  • HTML is the foundation of a webpage.
  • The body element represents the visible content shown to the user.
  • The head element contains information about the webpage.
  • The footer usually contains the author, copyright, contact, sitemap, and navigation.
  • Visual HTML elements are contained within opening and closing tags.
  • h1-h6 elements represent the level of heading for each text group.
  • p element represents a paragraph or block of text.
  • ul, ol and li elements represent unordered lists, ordered lists and list items, in the form of bulleted or numbered lists.
  • img element contains information about images that are displayed on the webpage.
  • src attribute defines the location of the image.
  • br element creates a line of empty space, or a line break between two blocks of content.

CSS

  • CSS = Cascading Style Sheets.
  • CSS defines the colors, aesthetics, visual layout, and organization of a webpage.
  • CSS styling techniques: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • A link element creates the connection to an external file.
  • The rel attribute specifies the relationshop between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource.
  • A margin indicates how much space we want around the outside of an element.
  • Border property sets an elements border
  • A padding indicates how much space we want around the content inside an element.
  • A . preceding a variable, designates this as a class selector therefore any HTML element assigned said variable will gain those properties.
  • Box-shadow adds a shadow effect around an elements frame.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add [file]: add a file as it looks now to your next commit (stage).
  • git commit -m "descriptive message": commit your staged content as a new commit snapshot.
  • git branch: list your branches.
  • git merge "branch": merge the specified branch's history into the current one.
  • git merge [alias]/[branch]: merge a remote branch into your current branch to bring it up to date.
  • git push [alias]/[branch]: transmit local branch commits to the remote repository branch.
  • git pull: fetch and merge any commits from the tracking remote branch.
  • pwd: displays the current working directory.
  • ls: displays the list of files and folders in the current directory.
  • cd: navigates between directories in the path.
  • cd ..: navigates backwards one level at a time.
  • mkdir: creates a new directory.
  • touch: creates a file inside a directory.

JavaScript

  • JavaScript is a high-level dynamic programming language that allows users to interact with websites.
  • Functions are a way of packaging functionality you wish to reuse.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A sequence of text is known as a string enclosed in single quote marks.
  • Numbers do not require quotes in JavaScript.
  • Boolean is a True/False statement also do not require quote marks.
  • An Array is a sequence that allows you to store mulitple values in a single reference.
  • Object in JavaScript can be anything and can be stored as a variable.
  • VAR is used to declare a function in JavaScript.
  • Conditionals are code structures used to test if an expression returns true or false, also known as an else...if.
  • Loops are used in JavaScript to perform repeated tasks based on a condition.
